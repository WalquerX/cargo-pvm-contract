# CLI Reference — `cargo pvm-contract`

## Overview

`cargo pvm-contract` is a Cargo subcommand for building and interacting with PolkaVM smart contracts targeting Polkadot's `pallet-revive`. It covers the full contract lifecycle: scaffolding, ABI encoding/decoding, and on-chain operations via native Substrate RPC.

### Installation

```bash
cargo install cargo-pvm-contract
```

### Top-level usage

```
cargo pvm-contract <SUBCOMMAND> [OPTIONS]
```

---

## Command Reference

### `init` — Scaffold a new contract project

Initialize a new PolkaVM contract project from a Solidity interface or an example.

**Usage**

```
cargo pvm-contract init [OPTIONS]
```

**Flags**

| Flag | Values | Description |
|------|--------|-------------|
| `--init-type` | `new`, `example` | Project type: blank contract or from a bundled example |
| `--example` | string | Name of the bundled example contract to use (implies `--init-type example`) |
| `--api-style` | `macro`, `dsl` | API style: proc macros (recommended) or builder DSL |
| `--allocator` | `bump`, `no-alloc` | Memory strategy: bump allocator or stack-only |
| `--name` | string | Project/crate name |
| `--sol-file` | path | Path to a `.sol` interface file for ABI scaffolding |

**Example**

```bash
# Interactive
cargo pvm-contract init

# Non-interactive: new project from a .sol file
cargo pvm-contract init \
  --init-type new \
  --api-style macro \
  --allocator no-alloc \
  --name my-token \
  --sol-file MyToken.sol

# From a bundled example
cargo pvm-contract init --init-type example --example mytoken
```

### `encode` — ABI-encode a function call or constructor

Encode function arguments into ABI-encoded hex calldata that can be passed to `instantiate --data` or `call --data`.

**Usage**

```
cargo pvm-contract encode --abi <PATH> [--function <NAME>] [ARGS...]
```

**Flags**

| Flag | Description |
|------|-------------|
| `--abi <PATH>` | Path to the ABI JSON file (generated by the build pipeline as `*.abi.json`) |
| `--function <NAME>` | Function name to encode. Omit to encode constructor arguments |
| `[ARGS...]` | Space-separated argument values to encode |

**Example**

```bash
# Encode a function call
cargo pvm-contract encode \
  --abi target/my_contract.release.abi.json \
  --function transfer \
  0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045 1000

# Encode constructor arguments
cargo pvm-contract encode \
  --abi target/my_contract.release.abi.json \
  1000000
```

### `decode` — Decode ABI-encoded calldata

Decode hex calldata back to human-readable argument values using the ABI.

**Usage**

```
cargo pvm-contract decode --abi <PATH> --data <HEX> [--constructor]
```

**Flags**

| Flag | Description |
|------|-------------|
| `--abi <PATH>` | Path to the ABI JSON file |
| `--data <HEX>` | Hex-encoded calldata to decode (0x-prefixed) |
| `--constructor` | Decode as constructor input (no 4-byte function selector prefix) |

**Example**

```bash
# Decode a function call
cargo pvm-contract decode \
  --abi target/my_contract.release.abi.json \
  --data 0xa9059cbb000000000000000000000000d8da6bf...

# Decode constructor data
cargo pvm-contract decode \
  --abi target/my_contract.release.abi.json \
  --data 0x00000000000000000000000000000000000f4240 \
  --constructor
```

### `upload` — Upload contract code to the chain

Upload a compiled `.polkavm` binary to the chain and obtain a code hash. Does not deploy a contract instance.

**Usage**

```
cargo pvm-contract upload --code <PATH> [--dry-run] [EXTRINSIC OPTIONS]
```

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--code <PATH>` | — | Path to the `.polkavm` contract binary |
| `--dry-run` | false | Simulate the upload without submitting to chain |

See [Common Options](#common-options) for `--url`, `--suri`, and `--storage-deposit-limit`.

**Example**

```bash
# Dry-run first to check for errors
cargo pvm-contract upload \
  --code target/my_contract.release.polkavm \
  --dry-run \
  --url ws://localhost:9944 \
  --suri //Alice

# Submit on-chain
cargo pvm-contract upload \
  --code target/my_contract.release.polkavm \
  --url ws://localhost:9944 \
  --suri //Alice
```

### `instantiate` — Deploy a contract

Upload the contract code and create a new contract instance in a single step. Returns the deployed contract address.

**Usage**

```
cargo pvm-contract instantiate --code <PATH> [OPTIONS] [EXTRINSIC OPTIONS]
```

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--code <PATH>` | — | Path to the `.polkavm` contract binary |
| `--data <HEX>` | — | ABI-encoded constructor arguments (0x-prefixed). Use `encode` to produce this |
| `--value <u128>` | 0 | Token value to transfer to the new contract on deploy |
| `--salt <HEX>` | — | Hex salt for deterministic address derivation (optional) |
| `--dry-run` | false | Simulate the deployment without submitting to chain |

See [Common Options](#common-options) for `--url`, `--suri`, and `--storage-deposit-limit`.

**Example**

```bash
# Instantiate without constructor args
cargo pvm-contract instantiate \
  --code target/my_contract.release.polkavm \
  --url ws://localhost:9944 \
  --suri //Alice

# Instantiate with constructor args (initial supply = 1_000_000)
CONSTRUCTOR=$(cargo pvm-contract encode \
  --abi target/my_contract.release.abi.json \
  1000000)

cargo pvm-contract instantiate \
  --code target/my_contract.release.polkavm \
  --data $CONSTRUCTOR \
  --url ws://localhost:9944 \
  --suri //Alice
```

### `call` — Call an existing contract

Send a transaction to call a function on a deployed contract. Use `--dry-run` for read-only queries.

**Usage**

```
cargo pvm-contract call --contract <H160> --data <HEX> [OPTIONS] [EXTRINSIC OPTIONS]
```

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--contract <H160>` | — | Contract address (0x-prefixed H160) |
| `--data <HEX>` | — | ABI-encoded calldata (0x-prefixed). Use `encode` to produce this |
| `--value <u128>` | 0 | Token value to transfer with the call |
| `--dry-run` | false | Simulate the call without submitting to chain (useful for view functions) |

See [Common Options](#common-options) for `--url`, `--suri`, and `--storage-deposit-limit`.

**Example**

```bash
# Encode the calldata first
CALLDATA=$(cargo pvm-contract encode \
  --abi target/my_contract.release.abi.json \
  --function transfer \
  0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045 1000)

# Dry-run a view call
cargo pvm-contract call \
  --contract 0xYourContractAddress \
  --data $CALLDATA \
  --dry-run \
  --url ws://localhost:9944 \
  --suri //Alice

# Submit a state-changing call
cargo pvm-contract call \
  --contract 0xYourContractAddress \
  --data $CALLDATA \
  --url ws://localhost:9944 \
  --suri //Alice
```

### `remove` — Remove uploaded contract code

Remove a previously uploaded contract code blob from the chain by its code hash. This frees storage and recovers the deposit.

**Usage**

```
cargo pvm-contract remove --code-hash <H256> [EXTRINSIC OPTIONS]
```

**Flags**

| Flag | Description |
|------|-------------|
| `--code-hash <H256>` | Code hash of the uploaded contract to remove (0x-prefixed H256) |

See [Common Options](#common-options) for `--url`, `--suri`, and `--storage-deposit-limit`.

**Example**

```bash
cargo pvm-contract remove \
  --code-hash 0xabc123... \
  --url ws://localhost:9944 \
  --suri //Alice
```

### `map-account` — Map a Substrate account to an EVM address

Map your Substrate (SS58) account to its corresponding EVM H160 address in `pallet-revive`. This is required before the account can interact with contracts on-chain.

**Usage**

```
cargo pvm-contract map-account [--dry-run] [EXTRINSIC OPTIONS]
```

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--dry-run` | false | Simulate the mapping without submitting to chain |

The account to map is derived from `--suri`. See [Common Options](#common-options).

**Example**

```bash
# Map //Alice to her EVM address
cargo pvm-contract map-account \
  --url ws://localhost:9944 \
  --suri //Alice

# Dry-run to preview
cargo pvm-contract map-account \
  --dry-run \
  --url ws://localhost:9944 \
  --suri //Alice
```

### `info` — Query on-chain contract metadata

Fetch and display on-chain metadata for a deployed contract (code hash, storage deposit, etc.).

**Usage**

```
cargo pvm-contract info --contract <H160> [--url <WS_URL>]
```

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--contract <H160>` | — | Contract address to query (0x-prefixed H160) |
| `--url <WS_URL>` | `ws://localhost:9944` | Websocket URL of the Substrate node |

**Example**

```bash
cargo pvm-contract info \
  --contract 0xYourContractAddress \
  --url ws://localhost:9944
```

### `rpc` — Execute a raw JSON-RPC call

Send a raw JSON-RPC request to the node. Useful for querying chain state or calling RPC methods not exposed by other subcommands.

**Usage**

```
cargo pvm-contract rpc --method <METHOD> [--url <WS_URL>] [PARAMS...]
```

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--method <METHOD>` | — | RPC method name (e.g. `system_chain`, `system_version`) |
| `--url <WS_URL>` | `ws://localhost:9944` | Websocket URL of the Substrate node |
| `[PARAMS...]` | — | RPC parameters as space-separated JSON values |

**Example**

```bash
# Query chain name
cargo pvm-contract rpc \
  --method system_chain \
  --url ws://localhost:9944

# Query node version
cargo pvm-contract rpc \
  --method system_version \
  --url ws://localhost:9944
```

### `account` — Query account balance and info

Display the balance and Substrate account ID for a given H160 address.

**Usage**

```
cargo pvm-contract account --addr <H160> [--url <WS_URL>] [--output-json]
```

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--addr <H160>` | — | H160 address to look up (0x-prefixed) |
| `--url <WS_URL>` | `ws://localhost:9944` | Websocket URL of the Substrate node |
| `--output-json` | false | Print output as JSON |

**Example**

```bash
cargo pvm-contract account \
  --addr 0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045 \
  --url ws://localhost:9944

# JSON output
cargo pvm-contract account \
  --addr 0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045 \
  --url ws://localhost:9944 \
  --output-json
```

## Common Options

The following flags are shared across all on-chain commands (`upload`, `instantiate`, `call`, `remove`, `map-account`):

| Flag | Default | Description |
|------|---------|-------------|
| `--url <WS_URL>` | `ws://localhost:9944` | Websocket URL of the Substrate node to connect to |
| `--suri <URI>` | `//Alice` | Secret key URI for signing transactions (e.g. `//Alice`, `//Bob`, or a mnemonic) |
| `--storage-deposit-limit <u128>` | — | Maximum storage deposit to allow (in planck). Prevents unexpectedly large deposits |

## End-to-End Example

Full workflow from project creation to on-chain interaction:

```bash
# 1. Scaffold a new contract project
cargo pvm-contract init \
  --init-type new \
  --api-style macro \
  --allocator no-alloc \
  --name my-token \
  --sol-file MyToken.sol

cd my-token

# 2. Build the contract (requires nightly + rust-src)
cargo +nightly build --release -Z build-std=core,alloc \
  --target riscv64emac-unknown-none-polkavm

# 3. Upload the bytecode to a local node
cargo pvm-contract upload \
  --code target/riscv64emac-unknown-none-polkavm/release/my-token.polkavm \
  --url ws://localhost:9944 \
  --suri //Alice

# 4. Encode constructor arguments (initial supply = 1_000_000)
CONSTRUCTOR=$(cargo pvm-contract encode \
  --abi target/riscv64emac-unknown-none-polkavm/release/my-token.abi.json \
  1000000)

# 5. Instantiate the contract
cargo pvm-contract instantiate \
  --code target/riscv64emac-unknown-none-polkavm/release/my-token.polkavm \
  --data $CONSTRUCTOR \
  --url ws://localhost:9944 \
  --suri //Alice
# → outputs contract address, e.g. 0xABCDEF...

CONTRACT=0xABCDEF...

# 6. Query contract info
cargo pvm-contract info \
  --contract $CONTRACT \
  --url ws://localhost:9944

# 7. Encode a transfer call
CALLDATA=$(cargo pvm-contract encode \
  --abi target/riscv64emac-unknown-none-polkavm/release/my-token.abi.json \
  --function transfer \
  0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045 500)

# 8. Dry-run the call first
cargo pvm-contract call \
  --contract $CONTRACT \
  --data $CALLDATA \
  --dry-run \
  --url ws://localhost:9944 \
  --suri //Alice

# 9. Submit the call
cargo pvm-contract call \
  --contract $CONTRACT \
  --data $CALLDATA \
  --url ws://localhost:9944 \
  --suri //Alice
```
